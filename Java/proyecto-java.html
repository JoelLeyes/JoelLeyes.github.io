Chat Empresarial JavaEE — Java EE | Servlets | Tomcat | Nginx | Docker | AWS ECR | EC2 | RDS PostgreSQL | MongoDB Atlas | Firebase FCM | GitHub Actions CI/CD

Solución web de mensajería interna orientada a empresas, diseñada para centralizar conversaciones, acelerar la coordinación entre equipos y dejar trazabilidad operativa. El proyecto integra un backend Java EE con arquitectura en capas, frontend servido por Nginx y despliegue contenerizado en AWS. Incluye autenticación, chats privados y grupales, reacciones, estado de escritura, notificaciones y un panel de administración, con foco en rendimiento, mantenibilidad y facilidad de despliegue.

Qué resuelve

Está pensado como un chat corporativo para organizaciones que necesitan comunicación interna segura, ordenada y extensible. La propuesta combina experiencia de usuario simple con una base técnica sólida: separación entre capa web, servicios y acceso a datos, además de un flujo de despliegue automatizable.

Arquitectura y alcance

  • Backend: Java EE con Servlets, JPA/Hibernate para persistencia, capa DAO y capa Service
  • Runtime: Apache Tomcat 10.x ejecutando el WAR dentro de contenedores Docker
  • Frontend: HTML, CSS y JavaScript servidos por Nginx (reverse proxy SSL)
  • Mensajería: chats privados y grupales, historial de mensajes, reacciones y typing indicator (TypingStore en memoria)
  • Notificaciones: notificaciones por email (SMTP / Gmail) y push mediante Firebase FCM
  • Auditoría y logs: MongoDB Atlas usado para almacenar logs y auditoría de acciones (MongoAuditService)
  • Almacenamiento de archivos: almacenamiento local en EC2 (carpeta `uploads`) servido por Nginx y accesible vía `FileDownloadServlet`
  • Seguridad: autenticación con JWT, CORS global (CorsFilter), HTTPS/TLS con certificado ZeroSSL
  • Despliegue y CI/CD: GitHub Actions construye imagen Docker, publica en AWS ECR; EC2 descarga y actualiza contenedores

Habilidades demostradas

  • Arquitectura en capas: separación clara entre DAO, Service, Servlet y modelos
  • Diseño de dominio: entidades para usuarios, chats, mensajes, reacciones, estados y roles
  • Experiencia de producto: panel de administración, login, conversaciones y feedback en tiempo real
  • Automatización: contenedorización con Docker y despliegue repetible en AWS
  • Operación cloud: ECR, EC2, SG, SSH y actualización remota por pipeline
  • Integraciones: MongoDB Atlas para auditoría, Firebase FCM para push y SMTP/Gmail para emails

Tecnologías y servicios

  • Backend: Java EE, Servlets, JPA/Hibernate, JWT
  • Servidor de aplicaciones: Apache Tomcat 10.x
  • Frontend: HTML5, CSS3, JavaScript, Nginx (reverse proxy y static assets)
  • DevOps: Docker, Docker Compose, GitHub Actions (build → tests → push a ECR)
  • AWS: ECR, EC2, Security Groups, RDS PostgreSQL (base relacional principal)
  • Almacenamiento: almacenamiento local en EC2 (`/uploads`) servido por Nginx
  • Logs / Auditoría: MongoDB Atlas (registro de auditoría y eventos)
  • Notificaciones: Firebase FCM (push) y SMTP/Gmail (email) — SMTP en puerto 587 con STARTTLS
  • Seguridad: HTTPS con ZeroSSL, CORS filter y JWT para protección de endpoints
  • Otras integraciones: Firebase, FCM, y servicios auxiliares para entrega de notificaciones
